# 005 AC30HWH NAM A2 First Render

Status: first pass complete

## Purpose

Render a real NAM A2 amp-head capture and compare it against Greybound Nox30.

This is the first end-to-end NAM reference pass:

- local `.nam` pack imported manually,
- source-safe NAM pack manifest generated,
- NAM A2 rendered offline through Python/Torch,
- Greybound rendered with matching sample rate and dry DI,
- metric report generated by `compare-wav`.

## Reference

NAM pack:

```text
lab/references/nam/AC30HWH/
```

Source-safe manifest:

```text
lab/references/nam/manifests/ac30hwh-6580.json
```

Selected model:

```text
TopBoost-Gain5.nam
```

Tone source:

```text
https://www.tone3000.com/tones/ac30hwh-6580
```

Model policy:

- NAM A2 only,
- amp-head capture,
- no cabinet in NAM,
- no Greybound IR in this first amp-head-only comparison.

## Renderer

The renderer is:

```text
lab/scripts/nam_a2_render.py
```

It uses the official Python `neural-amp-modeler==0.13.0` package in a temporary
Python 3.11 `uv run` environment. For A2 `SlimmableContainer` files, it selects
the highest-quality submodel, currently the submodel with the largest
`max_value`, then renders through NAM's WaveNet loader.

This is an offline lab path only. It is not part of the Greybound runtime.

## Input Preparation

Greybound requires the input WAV sample rate to match `--sample-rate`. NAM A2 is
48 kHz, so the dry sample was resampled to:

```text
lab/stimuli/teenager-electric-guitar-smooth-chords-dry_94bpm_G_major-48k.wav
```

The resampled file is ignored by git. The generator command should be promoted
to a reusable lab command if we keep using arbitrary musical DI references.

## Renders

NAM render:

```sh
make lab-render-nam \
  NAM_INPUT_WAV=lab/stimuli/teenager-electric-guitar-smooth-chords-dry_94bpm_G_major-48k.wav \
  NAM_INPUT_DB=-40 \
  NAM_OUTPUT_DB=-24 \
  NAM_OUTPUT_WAV=lab/references/nam/renders/ac30hwh-6580-topboost-gain5-input-40.wav \
  NAM_METADATA=lab/references/nam/renders/ac30hwh-6580-topboost-gain5-input-40.run.json
```

Greybound render:

```sh
uv --project lab run greybound-lab render-rig \
  --rig rigs/nox30-driven.json5 \
  --input-wav lab/stimuli/teenager-electric-guitar-smooth-chords-dry_94bpm_G_major-48k.wav \
  --output-wav lab/renders/nox30-driven-for-nam-topboost-gain5-input-40.wav \
  --metadata lab/renders/nox30-driven-for-nam-topboost-gain5-input-40.run.json \
  --render-seconds 20 \
  --sample-rate 48000 \
  --period-size 16 \
  --input-db -40 \
  --output-db -24
```

Comparison:

```sh
uv --project lab run greybound-lab compare-wav \
  --candidate lab/renders/nox30-driven-for-nam-topboost-gain5-input-40.wav \
  --reference lab/references/nam/renders/ac30hwh-6580-topboost-gain5-input-40.wav \
  --metadata lab/renders/nox30-driven-for-nam-topboost-gain5-input-40.run.json \
  --segments lab/segments/guitar-chords.markers.json \
  --report lab/reports/nox30-driven-vs-nam-ac30hwh-topboost-gain5-input-40.md \
  --max-lag-ms 200
```

## First Result

Level summary:

| Render | Peak dBFS | RMS dBFS |
| --- | ---: | ---: |
| NAM AC30HWH TopBoost Gain5 | -40.33 | -60.31 |
| Greybound Nox30 driven | -22.37 | -70.06 |

Alignment and global metrics:

- estimated candidate latency: `-17 samples` / `-0.354 ms`,
- candidate gain correction: `-8.197 dB`,
- null residual relative to reference: `-0.07 dB`,
- log-spectral distance: `23.06 dB`,
- envelope error: `-0.15 dB`.

Dominant local findings:

- Greybound's RMS is about `9.75 dB` lower than the NAM render, but its peak is
  much higher; this points to a transient/gain-staging mismatch, not just a
  static output gain mismatch.
- Local gain in sustain windows is around `+24 dB`, meaning Greybound has far
  less sustained energy than the NAM reference after alignment.
- Presence-band residual is positive in most musical windows, while mid-band
  residual is negative; this points toward tone-stack/power-stage voicing before
  cabinet IR is involved.
- Sag-window drop delta is `12.40 dB`; Greybound's dynamic level behavior is not
  tracking this NAM reference.
- High-band residual is measurable, but this musical DI is not the final aliasing
  test. Use controlled synthetic stimuli for aliasing decisions.

## Interpretation

This is a successful pipeline milestone, not yet a model-quality milestone.

The lab can now render a real NAM A2 amp-head capture and compare it against a
Greybound rig. The first comparison says the next model work should focus on
gain staging, sustain/compression, and dynamic response before trying to match
fine spectral details.

The provisional NAM input calibration is not final. The first useful pass used:

```text
NAM_INPUT_DB=-40
NAM_OUTPUT_DB=-24
```

Lower input settings made the NAM reference too quiet. Higher settings should be
explored with clipping checks before being accepted as the standard.
